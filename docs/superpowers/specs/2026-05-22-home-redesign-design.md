# 首页改版设计文档

**日期**: 2026-05-22
**状态**: 待确认
**设计方向**: A · iOS 原生风

## 1. 设计目标

解决当前首页四个核心问题：
- 信息密度太低（只有日期切换和交易列表）
- 视觉不够精致（毛玻璃风格不统一，卡片单调）
- 交互不够丰富（只有查看和删除）
- 整体风格太平庸（缺乏辨识度）

## 2. 页面布局（从上到下）

### 2.1 顶部区域
- 日期文字：16sp，iOS Blue 色，点击弹出日期选择器
- 大标题："今日收支"，28sp Bold

### 2.2 收支概览卡片（毛玻璃）
- 左侧：支出金额，32sp Bold，红色 (#FF3B30)
- 中间：竖线分隔
- 右侧：收入金额，32sp Bold，绿色 (#34C759)
- 底部：预算进度条（4dp 高，圆角）
  - 背景色：rgba(0,0,0,0.04)
  - 进度色：线性渐变 #FF3B30 → #FF6B6B
  - 文字："月预算 ¥X,XXX · 已用 XX%"，11sp，灰色

### 2.3 分类概览卡片（毛玻璃）
- 标题："分类概览"，14sp SemiBold
- 内容：3 个分类小卡片 + 1 个"查看全部"卡片，横向排列
  - 每个分类卡片：emoji(20sp) + 分类名(11sp) + 金额(14sp SemiBold)
  - 分类卡片背景色：对应分类的浅色（如餐饮 rgba(255,59,48,0.08)）
  - "查看全部"卡片：⋯ 图标 + "全部" + 箭头
- 数据来源：当日支出按分类汇总，取 Top 3

### 2.4 交易列表区域
- 标题："今日交易"，16sp SemiBold
- 毛玻璃卡片包裹的交易列表
- 每条交易：分类名 · 子分类 / 时间 · 渠道 / 金额

## 3. 新增数据需求

### 3.1 分类汇总查询
- DAO 新增方法：`getCategoryExpenseByDateRange(startTime, endTime): Flow<List<CategoryExpense>>`
- 现有 `CategoryExpense` 数据类可直接使用

### 3.2 月预算功能
- 新增 `Budget` 数据类（已存在）
- DAO 新增方法：`getBudgetByMonth(yearMonth: String): Flow<Budget?>`
- 首次使用默认预算 ¥3,000
- 预算设置入口：分类概览卡片的预算进度条区域，点击跳转设置

### 3.3 月支出汇总
- 复用现有 `getExpenseByDateRange`，传入当月首日和末日

## 4. 交互设计

### 4.1 日期选择
- 点击日期文字 → 弹出 DatePickerBottomSheet（已有）
- 选择日期后，收支概览、分类概览、交易列表同步更新

### 4.2 分类概览交互
- 点击分类卡片 → 筛选该分类下的交易列表
- 点击"查看全部" → 跳转报表页（后续版本实现）

### 4.3 预算进度条交互
- 点击预算进度条 → 跳转预算设置（后续版本实现）
- 当前版本：进度条仅展示，不可点击

### 4.4 交易列表交互
- 长按交易 → 弹出编辑/删除菜单
- 点击交易 → 弹出编辑浮窗（已有 EditTransactionSheet）

## 5. 动画

- 收支金额变化时：数字滚动动画（AnimateContent）
- 分类概览进入：fadeIn + slideUp，300ms
- 预算进度条：宽度动画，500ms，FastOutSlowInEasing

## 6. 不做的事（YAGNI）

- 暂不做暗色模式适配
- 暂不做趋势图（留给报表页）
- 暂不做预算设置页（仅展示进度条，默认 ¥3,000）
- 暂不做分类筛选交易列表（仅展示分类卡片）

## 7. 涉及文件

| 文件 | 改动 |
|------|------|
| `HomeScreen.kt` | 重写布局，新增分类概览和预算进度条 |
| `HomeViewModel.kt` | 新增 categoryExpenses 和 monthlyBudget 状态 |
| `TransactionDao.kt` | 新增 getCategoryExpenseByDateRange 查询 |
| `TransactionRepository.kt` | 新增分类汇总和预算相关方法 |
| `AppDatabase.kt` | 可能需要新增 budget 表（如果预算需要持久化） |

## 8. 风险

- 分类汇总查询在数据量大时可能较慢 → 已有 category 索引，且按日过滤数据量有限
- 预算功能如果需要持久化，需要新增数据库表 → 当前版本用 SharedPreferences 存储即可
