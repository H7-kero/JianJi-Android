优化建议

高 补充 HomeScreen / RecordBottomSheet UI 测试
当前 UI 测试仅覆盖 ReportScreen 的空状态。建议补充：首页今日收支卡片显示、交易列表渲染、记账弹窗打开/关闭、表单输入交互等 Compose UI 测试。UI 层覆盖率目前仅 ~15%，是最大的覆盖盲区。
高 金额输入边界值测试
RecordViewModel.saveTransaction() 对金额的验证较简单（toDoubleOrNull），建议增加：超大金额溢出、科学计数法输入（1e10）、前导零（007.5）、小数精度（0.1+0.2）等边界用例，防止异常金额进入数据库。
中 时间相关测试可重复性
Repository 和 ViewModel 中使用 Calendar.getInstance() 获取当前时间，导致测试依赖系统时钟。建议注入 Clock 接口，使时间可 mock，提升测试可重复性和确定性。
中 DatePickerBottomSheet / WheelPicker 测试
滚轮选择器和日期选择器是 v2 新增的核心交互组件，目前没有测试覆盖。建议至少补充：日期选择回调、"今日"按钮跳转、未来日期禁用等关键逻辑的单元测试。
低 GlassmorphismTheme / LiquidGlassShader 测试
液态玻璃着色器（AGSL）是纯视觉效果组件，测试价值较低。如果后续着色器逻辑变复杂（如参数化折射率），可考虑抽取纯函数做单元测试。
低 CI/CD 自动化测试集成
建议在 GitHub Actions 中配置 CI 流水线：每次 PR 自动运行单元测试（./gradlew test），合并到 main 时运行全量测试（含 androidTest），测试失败阻止合并。
